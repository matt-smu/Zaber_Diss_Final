    \subsection{Setup}\label{setup}

We can run boromir with the following command:

\texttt{./run\_boromir.py\ -\/-benchmarks=mttf\ -\/-benchmark\_config\_file=mttf.yml\ -\/-boromir\_run\_count=1000\ -\/-bigquery\_table=\textquotesingle{}cloud-performance-tool:test\_notebook\_bq.test1\textquotesingle{}}

mttf.yml:

\begin{verbatim}
mttf:
    flags:
        # secmet_score_strategy: 'reliability'
        secmet_plot_intermediate_graphs: False
        secmet_random_cvss_score: True
\end{verbatim}

Using the mttf.yml file helps us track runs with version control
systems, but all flags can be specified at the command line as well.
\texttt{-\/-secmet\_random\_cvss\_score=True} and
\texttt{-\/-boromir\_run\_count=1000} create 1000 samples with
vulnerability scores chosen from a uniform random distribution in
\texttt{{[}0.0,\ 10.0{]}} to represent a CVSS score. Other distributions
and intervals can be defined as well.

The flag
\texttt{-\/-bigquery\_table=\textquotesingle{}cloud-performance-tool:test\_notebook\_bq.test1\textquotesingle{}}
tells boromir where to publish the results. If no publishing target is
specified the results are stored as json in the \$TMP directory.
Influxdb setup is covered in notebook
\texttt{000\_influxdb\_setup.ipynb}.
